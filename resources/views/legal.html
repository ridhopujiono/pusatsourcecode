Dokumen Legal Pusat Source Code

Dokumen ini mengatur hak, kewajiban, dan batas tanggung jawab antara Pusat Source Code dan pembeli produk digital berupa source code aplikasi.

Terakhir diperbarui: 9 Maret 2026

Syarat & Ketentuan

Dengan melakukan pemesanan dan/atau pembayaran, Anda dianggap telah membaca, memahami, dan menyetujui ketentuan berikut:

  1. Seluruh produk yang dijual adalah produk digital berupa source code, dokumentasi, dan/atau file pendukung.
  2. Pembelian memberikan hak penggunaan sesuai lisensi standar untuk kebutuhan internal bisnis/proyek pembeli, kecuali dinyatakan lain secara tertulis.
  3. Pembeli dilarang menjual ulang, mendistribusikan ulang, membagikan, atau mengklaim kepemilikan penuh atas source code tanpa izin tertulis/lisensi khusus dari Pusat Source Code.
  4. Pusat Source Code berhak menolak layanan atau membatalkan transaksi yang terindikasi melanggar hukum, hak cipta, atau kebijakan internal.
  5. Pembeli wajib memastikan kebutuhan teknis (server, domain, dependensi, dan environment) sesuai spesifikasi produk sebelum melakukan pembelian.
  6. Dukungan teknis diberikan secara wajar sesuai ruang lingkup produk. Penambahan fitur di luar deskripsi produk dapat dikenakan biaya terpisah.
  7. Pusat Source Code tidak bertanggung jawab atas kerugian tidak langsung, kehilangan keuntungan, kehilangan data, atau dampak bisnis lain akibat implementasi source code di luar kontrol kami.
  8. Jika ditemukan bug, tim kami akan melakukan upaya perbaikan yang wajar. Namun, tanggung jawab kami terbatas pada perbaikan teknis source code sesuai ruang lingkup awal produk.
Untuk kebutuhan lisensi komersial khusus (misalnya white-label atau hak distribusi ulang), silakan ajukan kerja sama melalui WhatsApp resmi kami.

Kebijakan Refund

Karena seluruh produk kami berupa file digital yang dapat langsung diakses/diunduh setelah pengiriman, maka secara umum berlaku kebijakan No Refund (tidak ada pengembalian dana).

Pengecualian hanya berlaku apabila file yang dikirimkan terbukti rusak/corrupt dan tidak dapat diperbaiki atau diganti oleh tim kami dalam waktu maksimal 3 x 24 jam sejak laporan diterima lengkap.

  • Laporan kendala wajib disertai bukti yang relevan (video/screenshot/error log) agar proses verifikasi dapat dilakukan cepat.
  • Jika kendala berasal dari konfigurasi server, versi software, atau environment pembeli yang tidak sesuai, maka tidak termasuk kriteria refund.
  • Jika tim berhasil memperbaiki atau mengganti file dalam batas waktu 3 x 24 jam, maka refund tidak dapat diajukan.
Kami berkomitmen menyelesaikan kendala teknis secepat mungkin agar source code dapat digunakan dengan baik di sisi pembeli.
